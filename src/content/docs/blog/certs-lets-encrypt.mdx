Let's Encrypt es una autoridad de certificación gratuita, automatizada y abierta que proporciona certificados SSL/TLS para cifrar las comunicaciones web. En este tutorial, aprenderemos a configurar Let's Encrypt para asegurar nuestras aplicaciones web.

## ¿Por qué usar Let's Encrypt?

Let's Encrypt ofrece certificados SSL gratuitos y fáciles de configurar, lo que permite a cualquier persona habilitar HTTPS en sus sitios web sin costo alguno. Esto es crucial para proteger la privacidad de los usuarios y mejorar la seguridad general de las aplicaciones web.

## Requisitos previos

Antes de comenzar, asegúrate de tener:

- Un dominio registrado y apuntando a tu servidor.

## Instalación de Certbot

Para obtener un certificado de Let's Encrypt, utilizaremos Certbot, una herramienta que automatiza el proceso de obtención y renovación de certificados.

```bash
sudo apt install certbot
```

## CSR (Certificate Signing Request)

Antes de solicitar un certificado, debes generar una CSR (Certificate Signing Request). Este archivo define los parametros del certificado y contiene información como el nombre del dominio, la organización y la ubicación.

1. Crea un archivo de configuración para la CSR:

```bash
sudo nano codedesignplus.com.cnf
```

2. Agrega el siguiente contenido al archivo:

```ini
[req]
default_bits = 2048
prompt = no
default_md = sha256
req_extensions = req_ext
distinguished_name = dn

[dn]
C = CO
ST = Bogota
L = Puente Aranda
O = codedesignplus
OU = IT
CN = *.codedesignplus.com

[req_ext]
subjectAltName = @alt_names

[alt_names]
DNS.1 = *.codedesignplus.com
DNS.2 = codedesignplus.com
```

## Generar la clave privada

La clave privada sera usada para firmar el CSR y el CSR sera enviado a la autoridad de certificación para obtener el certificado. 

```bash
openssl genrsa -out codedesignplus.com.key 2048
```

## Generar el CSR

Este comando generará el CSR usando la clave privada y el archivo de configuración que creamos anteriormente:

```bash
openssl req -new -key codedesignplus.com.key -out codedesignplus.com.csr -config codedesignplus.com.cnf
```

## Asignación de permisos

Asegúrate de que los permisos del archivo de configuración y la clave privada sean correctos para evitar problemas de seguridad:

```bash
chmod 644 codedesignplus.com.cnf 
chmod 600 codedesignplus.com.key
```

## Solicitar el certificado

Ahora que tenemos el CSR, podemos solicitar el certificado a Let's Encrypt usando Certbot:

```bash
sudo certbot certonly --manual --preferred-challenges dns --csr codedesignplus.com.csr -d "*.codedesignplus.com" -d "codedesignplus.com"
```
